Advanced Database Objects
Trigger trg_after_appointment_insert

Fires automatically every time a new Appointment is inserted. It creates a corresponding Unpaid Billing row ($0.00, due date = scheduled date + 30 days) without any application code. Because appointments can be created from many entry points — the UI, a migration script, or a DBA running SQL directly — placing this logic in the database guarantees billing consistency no matter how the appointment was created. 

AFTER INSERT ON Appointment FOR EACH ROW
  IF NOT EXISTS (SELECT 1 FROM Billing WHERE appointmentID = NEW.appointmentID)
    INSERT INTO Billing (totalAmount, billingDate, dueDate, paymentStatus, appointmentID)
    VALUES (0.00, DATE(NEW.scheduledDateTime), DATE_ADD(..., INTERVAL 30 DAY), 'Unpaid', NEW.appointmentID)
Stored Procedure sp_complete_appointment(appointmentID, actualStart)

Completing a visit requires updating two tables at once: marking the Appointment as Completed and the Queue entry as Served. Wrapping both UPDATEs in a stored procedure ensures they always execute together atomically, preventing a state where an appointment is completed but the queue entry is still showing as Waiting. 

CALL sp_complete_appointment(appointmentID, actualStart)
  → UPDATE Appointment SET status='Completed', actualStartTime=actualStart
  → UPDATE Queue SET queueStatus='Served', calledTime=actualStart
Function fn_wait_duration(appointmentID, queuePos)

Takes an appointmentID and queuePos, and returns the wait duration in minutes. If the patient is still waiting (calledTime is NULL), it calculates elapsed time using NOW(); once the patient has been called, it uses the actual calledTime instead. 

fn_wait_duration(p_appointmentID, p_queuePos) → INT (minutes)
  IF calledTime IS NULL → TIMESTAMPDIFF(MINUTE, arrivalTime, NOW())
  ELSE                  → TIMESTAMPDIFF(MINUTE, arrivalTime, calledTime)
View v_waiting_patients

Joins five tables together and filters to only patients currently with queueStatus = 'Waiting', including their department, queue position, and arrival time. The queue page can simply SELECT * FROM v_waiting_patients to get a live waiting list without rewriting the complex JOIN every time. 

Queue → Appointment → Room → Department → Person
WHERE queueStatus = 'Waiting'
→ returns: personID, firstName, lastName, deptName, queuePos, arrivalTime
